# OsuStocks.Web

Next.js 16 frontend for the OsuStocks API (an osu! stock-market game). See
[README.md](README.md) for setup, configuration, and deployment.

## Task tracking

All project work is tracked in [TASKS.md](TASKS.md). Treat it as the source of
truth for what's planned, in progress, and finished.

- **Read `TASKS.md` at the start of a session** to see current priorities and
  avoid redoing finished work.
- **Keep it current as part of your changes** — move tasks through the sections:
  - New ideas or out-of-scope finds → **Backlog**
  - Planned for the near term → **To Do**
  - Started working on it → **In Progress**
  - Finished _and verified_ → **Done (Archived)**, dated, newest first
- One line per task, use `- [ ]` / `- [x]` checkboxes, and update `TASKS.md` in
  the same commit as the work it describes so the file and code stay in sync.
